# Рыбалка — Telegram-бот-игра

Telegram-бот с мини-игрой про рыбалку.

Пользователь открывает кнопки с волнами и собирает улов: рыб, золотую рыбку, старый башмак или пустую попытку.

## Возможности

- стартовое сообщение с картинкой;
- игровое поле с inline-кнопками;
- 5 попыток на одну игру;
- одно меняющееся сообщение вместо цепочки новых сообщений;
- иллюстрации и описания для каждого результата;
- кастомные эмодзи на уже открытых кнопках;
- отдельная механика для золотой рыбки;
- кнопка «Сыграть ещё раз» после окончания попыток;
- кнопка «Похвастаться уловом»;
- генерация картинки с уловом для шеринга;
- простая внутренняя статистика.

## Технологии

- Python;
- python-telegram-bot;
- python-dotenv;
- Pillow.

## Структура проекта

`bot.py` — основной файл с логикой бота.

`requirements.txt` — зависимости проекта.

`.env.example` — пример файла с переменными окружения.

`assets/` — изображения и визуальные материалы бота.

## Настройка

Создайте файл `.env` рядом с `bot.py`:

```text
BOT_TOKEN=ваш_токен_бота
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

Запустите бота:

```bash
python bot.py
```

## Команды

`/start` — запуск игры.

`/stats_admin_fp` — статистика по игре.

## Синхронизация с GitHub

Файл `sync_to_github.command` сохраняет текущую версию бота и отправляет ее в GitHub.

Его можно запускать двойным кликом после правок.

В GitHub отправляются код и картинки. Не отправляются:

- `.env` с токеном бота;
- `stats.sqlite` со статистикой;
- `work/photo_file_ids.json` с быстрыми адресами картинок;
- временные файлы из `work/` и `outputs/`.

Эти файлы нужно переносить на рабочий сервер отдельно.
